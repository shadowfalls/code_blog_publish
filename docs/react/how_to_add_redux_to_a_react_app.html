Back in the old days, when html was rendered by the server using languages like PHP, there was very little, if not any, control over the state or session of the application happening in the frontend. When we have to go to a different page we had to request the server, and the server sends back the HTML and the entire page refreshes.

Then came the time when frontend frameworks like React became popular and suddenly, the frontend had more control over the application's state. We didn't have to reload the entire page to go to a different page in your application. When we type something in a form React keeps it in its state.

But when your application starts to get bigger you will find a need to share some data between two different pages or components. This is where tools like Redux are being used. Redux is an amazing tool that allows you to keep all your application's state in one centralized store. The Redux store handles each component's state.

Basics of state management with Redux:

In this tutorial we will be adding Redux to a simple React application. You can create a simple React app using the commandnpx create-react-app <application name>. You can read more about it in React's official documentation.

A simple Redux integration has the following parts:

Store: It holds the state of the application

Actions: These are functions that can be called from within a component that will operate on the store.

Slice: This is where we set the initial state of the application. It also has a reducers section with actions that can be used to manipulate the data in the store.

It will also make it harder to write unit tests for this component. There is a better way to pass the actions and the store state to the Incrementer component. The connect() accepts two functions as parameters, which are mapStateToProps and mapDispatchToProps. mapStateToProps as the name suggests can be used to pick the state variables that component needs and pass them as props to the component. mapDispatchToProps will pass the selected actions encapsulated by the dispatch method as props to the component. This is a container component. It is a good practice to send data and actions to all the components through only the container component.
